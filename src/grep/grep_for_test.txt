-e PATTERN: Задает шаблон для поиска. Этот флаг можно использовать несколько раз для задания нескольких шаблонов. Например, grep -e pattern1 -e pattern2 file ищет строки, соответствующие pattern1 или pattern2.

-i: Игнорирует регистр символов. Поиск будет нечувствителен к регистру, например, grep -i "pattern" file найдёт как "Pattern", так и "pattern".

-v: Инвертирует совпадения. Вместо того, чтобы выводить строки, содержащие шаблон, выводит строки, не содержащие его. Например, grep -v "pattern" file выведет все строки, которые не содержат "pattern".

-c: Выводит количество строк, содержащих шаблон. Вместо вывода самих строк, содержащих шаблон, выводит только число таких строк. Например, grep -c "pattern" file может вернуть 5, если найдено 5 строк.

-l: Выводит только имена файлов, содержащих шаблон. Если шаблон найден в файле, будет выведено только имя файла, а не сами строки. Например, grep -l "pattern" file1 file2 выведет file1 и/или file2, если шаблон найден в них.

-n: Выводит номера строк, содержащих шаблон. Вместе с каждой строкой, содержащей шаблон, выводится ее номер. Например, grep -n "pattern" file может вывести 3:pattern found here, если шаблон найден в строке 3.

-h: Подавляет вывод имен файлов. Полезно, если grep используется с несколькими файлами, но вы хотите видеть только сами строки без имен файлов. Например, grep -h "pattern" file1 file2.

-s: Подавляет сообщения об ошибках о несуществующих или нечитаемых файлах. Например, grep -s "pattern" nofile.txt не выведет сообщение об ошибке.

-o: Выводит только совпавшие части строк, а не все строки. Например, grep -o "pattern" file выведет только "pattern" для каждой строки, содержащей этот шаблон.

-f FILE: Считывает шаблоны из файла. Каждый шаблон в файле используется как отдельный шаблон для поиска. Например, если patterns.txt содержит строки pattern1 и pattern2, то grep -f patterns.txt file будет искать pattern1 и pattern2.
